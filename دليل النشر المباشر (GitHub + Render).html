الفكرة في سطر

نرفع كود التطبيق إلى GitHub (مستودعك)، ثم نربطه بخدمة استضافة Render التي تشغّله وتعطيك رابطاً عاماً دائماً مثل https://lu-leaders-live.onrender.com. تفتحينه على شاشة القاعة، ويدخل الطلاب من هواتفهم بمسح رمز QR. لا تبدأ اللعبة إلا بضغطك على زر «ابدأ اللعبة» بعد تأكدك من انضمام الجميع.

الجزء الأول — الرفع إلى GitHub (مرة واحدة)

1) سجّلي الدخول إلى github.com ← زر New ← أنشئي مستودعاً جديداً فارغاً باسم lu-leaders-live (Public أو Private)، دون تعليم README، ثم Create، وانسخي رابطه المنتهي بـ .git.
2) على جهازك، افتحي المجلد: C:\Users\faisa\lu_live وانقري نقراً مزدوجاً على push.bat، ثم الصقي رابط المستودع عند الطلب. (سيفتح المتصفح لتسجيل الدخول إلى GitHub أول مرة — وافقي.)
يتطلب وجود Git على الجهاز (مجاني من git-scm.com). إن لم يكن مثبتاً، ثبّتيه ثم أعيدي تشغيل push.bat. بدّلي الأمر داخل push.bat ليس ضرورياً — هو يسألك عن الرابط فقط.

الجزء الثاني — النشر على Render (مرة واحدة)

3) ادخلي render.com وأنشئي حساباً مجانياً (يمكن الدخول بحساب GitHub مباشرة).
4) New ← Web Service ← Build and deploy from a Git repository، واختاري مستودع lu-leaders-live.
5) سيكتشف Render أنه Node تلقائياً (Build: npm install · Start: node server.js · Plan: Free). اضغطي Create Web Service وانتظري دقيقتين.
6) ستحصلين على الرابط العام الدائم أعلى الصفحة. هذا رابط اللعبة. افتحيه على شاشة القاعة ← أنا المحاضِرة — تسجيل الدخول ← أدخلي اسم المستخدم وكلمة المرور ← يظهر رمز QR والرمز الرقمي للطلاب.
مهم عن الباقة المجانية: تنام الخدمة المجانية بعد ١٥ دقيقة خمول وتستيقظ خلال ~٥٠ ثانية. لذلك افتحي الرابط قبل المحاضرة بدقيقتين ليستيقظ. لضمان جاهزية فورية طوال أسبوع الامتحان، رقّي الخدمة إلى Starter (٧ دولار/شهر) ثم ألغيها بعد الأسبوع. (Render يدعم WebSockets التي يحتاجها التطبيق.)

دخول المحاضِرة (اسم مستخدم وكلمة مرور) 🔐

لمنع الطلاب من بدء اللعبة أو نشرها، أصبح بدء أي غرفة يتطلّب تسجيل دخول المحاضِرة فقط. بيانات الدخول مُخزّنة على الخادم (لا تظهر للطلاب أبداً) ويمكنك تغييرها من Render.

القيمة الافتراضيةالتغيير من Render
اسم المستخدم: iman
كلمة المرور: Leaders2026		من صفحة الخدمة في Render ← تبويب Environment ← Add Environment Variable أضيفي مفتاحاً باسم ADMIN_PASSWORD وقيمته كلمة مرور سرية تختارينها (واختيارياً ADMIN_USER لاسم مستخدم آخر) ← Save Changes. سيُعاد النشر تلقائياً.
غيّري كلمة المرور الافتراضية قبل أسبوع الامتحان عبر ADMIN_PASSWORD في Render، ولا تشاركيها مع الطلاب.

كيف «يسجّل» الطلاب الدخول؟

كما في Kahoot تماماً: لا كلمات مرور ولا حسابات. يمسح الطالب رمز QR (أو يفتح الرابط ويُدخل الرمز الرقمي المكوّن من ٤ أرقام)، ثم يكتب اسمه. الرمز الرقمي للغرفة هو ما يحمي جلستك — لا يدخل لعبتك إلا من يملك الرمز/الرمز المعروض على شاشتك.

أفضل ممارسات التوقيت (مطبّقة في التطبيق)

المرحلةالتوصية والتطبيق
غرفة الانتظارلا تبدأ اللعبة تلقائياً أبداً. تظهر أسماء المنضمّين فوراً على شاشتك، وتبدئين أنتِ عند جاهزية الجميع بزر «ابدأ اللعبة» (مُعطّل حتى ينضم طالب واحد على الأقل).
قبل أول سؤالعدّ تنازلي مُتزامن ٣–٢–١ على كل الأجهزة لينتبه الطلاب وينظروا للشاشة.
زمن السؤالافتراضياً ٢٥ ثانية، وتحدّدينه أنتِ من غرفة الانتظار (من ٥ إلى ١٨٠ ثانية) في خانة «زمن السؤال (ثانية)». الأسرع في الإجابة الصحيحة يحصل على نقاط أعلى.
كشف الإجابةتظهر الإجابة الصحيحة + تفسير تعليمي للإجابة (لماذا هي الصحيحة) + أسرع ١٠ طلاب + المتصدّر — على شاشتك وعلى هاتف كل طالب. وتتقدّمين أنتِ بزر «السؤال التالي» — وهي لحظة المناقشة الصفّية قبل الانتقال.
التوقيت محسوب على الخادم (server-authoritative) فيكون عادلاً للجميع بغضّ النظر عن سرعة الإنترنت لدى كل طالب.

تحديث المحتوى لاحقاً (والتعديل أثناء التجارب)

عدّلي الأسئلة والإجابات وعمود «تفسير الإجابة» في ملف Excel، ثم استورديها من زر «استيراد أسئلة (Excel)» في غرفة الانتظار قبل بدء اللعبة — دون أي برمجة، وفي أي تجربة أو محاضرة. ولأي تعديل على الكود، يكفي تشغيل push.bat مرة أخرى؛ سيُحدّث Render النشر تلقائياً.

إذا لم يتحدّث النشر تلقائياً بعد رفع تعديل: من صفحة الخدمة في Render اضغطي Manual Deploy ← Deploy latest commit، وتأكّدي في Settings ← Build & Deploy أن Auto-Deploy = Yes وأن المستودع المربوط هو faisalhosny/lu-leaders-live على فرع main.
